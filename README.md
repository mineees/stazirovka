## Неделя 1 — Основы Linux, файрвол, диагностика

### Задание 1 — Базовый файрвол на iptables

**Сложность:** простая  
**Темы:** iptables, Linux

#### Описание

Настроить политику по умолчанию DROP для цепочки INPUT. Разрешить только нужный трафик: SSH (22), HTTP (80), HTTPS (443) и loopback. Правила сохранить и добавить в автозагрузку.

#### Что должно быть сделано

- [ ] Политика по умолчанию `DROP` для `INPUT`
- [ ] Разрешён трафик: SSH (22), HTTP (80), HTTPS (443), loopback (`lo`)
- [ ] Разрешены `ESTABLISHED` и `RELATED` соединения
- [ ] Правила сохранены через `iptables-save > /etc/iptables/rules.v4`
- [ ] Автозагрузка правил настроена (через `iptables-restore` или `netfilter-persistent`)
- [ ] Написан отчёт: вывод `iptables -L -n -v` с пояснениями

### Задание 2 — Bash-скрипт резервного копирования

**Сложность:** простая  
**Темы:** Linux, Bash

#### Описание

Написать скрипт, который архивирует указанные директории, сохраняет архив в `/backup/YYYY-MM-DD/` и удаляет бэкапы старше 7 дней. Запуск по расписанию через cron.

#### Что должно быть сделано

- [ ] Скрипт принимает список директорий как аргументы или из конфига
- [ ] Архив сохраняется в `/backup/$(date +%Y-%m-%d)/`
- [ ] Бэкапы старше 7 дней удаляются автоматически
- [ ] Все действия логируются в `/var/log/backup.log` с временными метками
- [ ] Cron-задача запускает скрипт каждую ночь в 02:00
- [ ] Скрипт корректно обрабатывает ошибки (директория не существует и т.п.)
