## Основы Linux, файрвол, диагностика

### Задание 1 — Настройка рабочего места

**Темы:** Linux, Bash

#### Описание

Получить машину Создать нового пользователя и полностью настроить его для работы вручную, без скриптов.

#### Что должно быть сделано

- [ ] Создан новый пользователь
- [ ] Пользователь добавлен в группу `sudo`
- [ ] Установлен и настроен `zsh`
- [ ] Установлен `oh-my-zsh` с минимум 2 плагинами (например, `git`, `zsh-autosuggestions`)
- [ ] Создан и настроен `~/.zshrc` (алиасы, PATH, тема) 
- [ ] Сгенерирована SSH-пара ключей
- [ ] Установлен Docker и Docker Compose
- [ ] Пользователь добавлен в группу `docker` (без sudo для docker-команд)
- [ ] Проверка: `docker run hello-world` работает от нового пользователя без sudo

### Задание 2 — Автоматизация настройки рабочего места (Bash-скрипт)

#### Описание

Написать bash-скрипт, который автоматически воспроизводит всё что было сделано руками в задании 1. Скрипт должен запускаться и полностью настраивать нового пользователя.

#### Что должно быть сделано

- [ ] Скрипт принимает имя нового пользователя как аргумент: `./setup.sh username`
- [ ] Создаёт пользователя, добавляет в `sudo` и `docker`
- [ ] Устанавливает Docker и Docker Compose (официальный способ)
- [ ] Устанавливает `oh-my-zsh` и базовый `.zshrc` с алиасами и плагинами
- [ ] Устанавливает `zsh` как shell по умолчанию для нового пользователя
- [ ] Все шаги логируются в файл с временными метками
- [ ] Скрипт корректно обрабатывает ошибки: если пользователь уже существует — сообщает об этом и не падает
- [ ] В конце выводит итог: что установлено, версии Docker и zsh

#### Что должно быть сделано

- [ ] Файл `docker-compose.yml` с сервисами `nginx` и `postgres`
- [ ] Внутренняя сеть `bridge` между контейнерами
- [ ] Named volume для данных PostgreSQL
- [ ] Bind mount для конфига nginx
- [ ] Наружу проброшен только порт 80
- [ ] `docker compose up -d` поднимает стек без ошибок

## Безопасность и мониторинг

### Задание 7 — Защита от brute-force: iptables + fail2ban
 
**Темы:** iptables, безопасность, Linux

#### Описание

Настроить комплексную защиту SSH от перебора паролей. Сначала настроить ограничения на уровне iptables, затем установить и настроить fail2ban для автоматической блокировки атакующих IP.

#### Что должно быть сделано

**iptables:**
- [ ] Ограничение новых подключений на порт 22: не более 5 за 60 секунд с одного IP (модуль `recent` или `hashlimit`)
- [ ] Превышение лимита — пакеты дропаются с логированием (`LOG --log-prefix "SSH_BRUTEFORCE: "`)
- [ ] Правила сохранены и добавлены в автозагрузку

**fail2ban:**
- [ ] Установлен и запущен `fail2ban`
- [ ] Настроен jail для SSH в `/etc/fail2ban/jail.local`: порог 5 попыток, бан на 1 час
- [ ] Настроено логирование заблокированных IP
- [ ] Проверка: `fail2ban-client status sshd` показывает активный jail
- [ ] Симуляция атаки (несколько неудачных SSH-подключений) — IP попадает в бан
- [ ] Написан отчёт: чем отличается защита через iptables от fail2ban, когда что использовать
